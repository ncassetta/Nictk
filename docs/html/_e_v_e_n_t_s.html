Events

In tkinter we have two ways to connect a callback to a widget event:

  • The bind() method, which allows the user to freely bind an event generated by the widget to a callback function.
  • The command option of the config() method which allows to bind a predefined event (the click for a button, the return for an Entry, ecc) to a callback function. The two methods differ: with bind() the callback gets a tkinter.Event as paramether, i.e. an object with various informations on what is happened, while with command the callback gets no parameter. So tkinter suggest to write for the callbacks the signature:
    def callback(event=None):

If you are not familiar with tkinter events, I suggest the following documentation:

  • Event types
  • Event class attributes
  • The bind(), bind_class() and bind_all() methods
  • How to specify the event to bind

In Nictk I made these changes:

  • Both the bind() method and the command option pass a tkinter.Event object to the callback. In the case of the command the event is a <<VirtualEvent>> and has only the following attributes:
    • .widget
    • .x
    • .y
    • .value (see below)
  • I added the command option to some other widgets: this is a list of the widgets which support it:
    • Nictk.Button (when the button is pressed)
    • Nictk.Checkbutton (when the button changes its state)
    • Nictk.Combobox (when an item is selected)
    • Nictk.Entry (when Return is pressed)
    • Nictk.Listbox (when an item is selected)
    • Nictk.Radiobutton (when the button is selected)
    • Nictk.Scale (when the scale index changes)
    • Nictk.Scrollbar (when the scrollbar is moved)
    • Nictk.Spinbox (when an item is selected, with arrow buttons or with Return)
  • For the command option you have the following choices:
    • command=_callback_ this only binds the callback to the event
    • command=(callback, value) setting a tuple callback - value the Event passed to the callback will contain the given value in the .value attribute (otherwise this will be set to None)

Examples of these procedures are in all example files.
